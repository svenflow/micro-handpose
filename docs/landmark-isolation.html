🔬 Landmark Model Isolation Test

Bypasses our palm detection — uses MediaPipe's detected hand position to crop, then runs our landmark model on that crop.

Purpose: Our full pipeline has ~1% landmark error vs MediaPipe. The activation-level debug showed the palm detection middle_MCP keypoint is 2.82% off, cascading into rotation and crop errors. This test isolates the landmark model by:
1. Running MediaPipe to get its final landmarks (ground truth)
2. Using MediaPipe's wrist + middle_MCP landmarks as if they were palm detection keypoints
3. Computing the crop ROI using our detectionToPixelROI math
4. Feeding that crop into our landmark model
5. Comparing to MediaPipe's landmarks

Expected: If our landmark model is correct, error should drop from ~1% to ~0.3% (matching MP GPU↔CPU variance). If it stays at ~1%, the landmark model architecture or weights also diverge.
